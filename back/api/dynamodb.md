## Users

### ハッシュキーとソートキーの選定

ユーザーの人数はあまり多くないため、id のみで設計する。何かで分割する必要ができた場合にソートキーを追加する

- hashkey

  - id

- attribute

  - name(for login)
  - display_name
  - password
  - is_manager

## Shifts

> ### パーティションとは、、、
>
> > シフトの時間をいくつかの時間区分に分けた時の時間単位のこと

パーティションごとのデータの追加が多いのでその処理がなるべく少ないクエリでいいように設計する

> ### 尚、重複した主キーを指定して挿入（Put）操作を行った場合、既存のアイテムが上書きされる

### キーの選定

- hashkey

  > ID をただ作成するよりも年や月、日を結合した文字列や数字を ID として定義したい

  - date+partition(文字列として結合)

- sortkey

  - user_id

### Example

```json
[
  {
    "partition": "2023-06-29-B",
    "userID": 0
  },
  {
    "partition": "2023-06-29-C",
    "userID": 0
  },
  {
    "partition": "2023-06-29-C",
    "userID": 1
  },
  {
    "partition": "2023-06-29-D",
    "userID": 1
  }
]
```
